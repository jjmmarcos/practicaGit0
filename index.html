Diferencias entre Sistema de control de versiones locales, centralizados y distribuidos.

El sistema de control de versiones locales consiste en copiar los archivos a otro directorio. Es muy común porque es muy simple, pero también muy propenso a errores.

El sistema de control de versiones centralizados tienen un único servidor, que contiene todos los archivos versionados, y varios clientes que descargan los archivos desde ese lugar central. Desde hace muchos años éste ha sido el estándar para el control de versiones.

Qué es GitHub y cuáles son sus utilidades.

Es un sistema de gestión de proyectos y control de versiones de código, como una red social para desarrolladores de software.

Brinda herramientas útiles para el trabajo en equipo dentro de un proyecto. Además es posible contrubuir para mejorar el software de los demás.
